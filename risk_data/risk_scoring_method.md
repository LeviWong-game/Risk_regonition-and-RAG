# 风险评分重建方法说明

## 1. 目标

本方法用于替代 `risk_data` 中原有的“经验分”，把风险分值改造成可解释的评分卡。输出字段固定为：

- `risk_score`：`0-100`
- `risk_level`：`L1-L4`
- `P`：升级概率
- `I`：影响后果
- `D`：处置难度/可控性缺口
- `trigger_flags`：升级触发项
- `reason_text`：解释文本

等级阈值保持兼容：

- `L1`: `0-25`
- `L2`: `26-50`
- `L3`: `51-75`
- `L4`: `76-100`

这套边界是闭区间，`25/50/75` 保持在较低一级，避免边界抖动。

## 2. 核心公式

评分公式：

```text
risk_score = 100 × (0.75 × P × I + 0.25 × D)
```

其中：

- `P` 反映纠纷升级的可能性
- `I` 反映纠纷一旦失控带来的影响后果
- `D` 反映单靠常规调解是否难以及时控制

`P/I/D` 都由二级指标加权得到，取值范围都是 `[0,1]`。

## 3. 指标体系

### 3.1 一级维度

| 维度 | 含义 | 二级指标 |
|---|---|---|
| `P` | 升级概率 | 冲突强度、持续性/复发性、涉事主体复杂度、升级触发信号 |
| `I` | 影响后果 | 涉及范围、安全/健康损害、经济/民生损失、公共秩序/舆情影响 |
| `D` | 处置难度 | 法律与事实争议度、跨部门协调难度、调解资源与时效压力、脆弱群体/紧迫性 |

### 3.2 默认二级权重

| 维度 | 二级指标 | 权重 |
|---|---|---|
| `P` | 冲突强度 | `0.35` |
| `P` | 持续性/复发性 | `0.25` |
| `P` | 涉事主体复杂度 | `0.20` |
| `P` | 升级触发信号 | `0.20` |
| `I` | 涉及范围 | `0.20` |
| `I` | 安全/健康损害 | `0.35` |
| `I` | 经济/民生损失 | `0.20` |
| `I` | 公共秩序/舆情影响 | `0.25` |
| `D` | 法律与事实争议度 | `0.25` |
| `D` | 跨部门协调难度 | `0.35` |
| `D` | 调解资源与时效压力 | `0.20` |
| `D` | 脆弱群体/紧迫性 | `0.20` |

### 3.3 打分锚点

每个二级指标统一按 `1-5` 档评分，再映射为：

| 原始档位 | 归一化值 |
|---|---|
| `1` | `0` |
| `2` | `0.25` |
| `3` | `0.5` |
| `4` | `0.75` |
| `5` | `1` |

锚点示例：

| 指标 | `1分` | `3分` | `5分` |
|---|---|---|---|
| 冲突强度 | 普通投诉或一般分歧 | 争执、对峙或一方拒绝配合 | 肢体冲突、暴力威胁或极端行为 |
| 持续性/复发性 | 首次发生 | 已多次反映或持续一段时间 | 长期积累且持续升级 |
| 安全/健康损害 | 无现实风险 | 存在明显健康、污染或轻伤风险 | 重伤、火灾、爆炸或严重公共安全风险 |
| 跨部门协调难度 | 单一主体可处理 | 需调解员或街道协同 | 需公安、法院、应急等强制或专业介入 |

完整锚点已经写入 [risk_scoring.py](/e:/风险预测和评估/risk_data/risk_scoring.py)。

## 4. 升级触发规则

连续分不是唯一依据，还要叠加红旗规则：

- 出现以下任一项，最低不低于 `L3`
  - 已发生伤害或明显暴力威胁
  - 现实公共安全隐患
  - 聚集上访或舆情扩散
- 同时出现两项以上，或已经出现公安/法院/司法强制介入时，最低不低于 `L4`

这部分逻辑已经在 [risk_scoring.py](/e:/风险预测和评估/risk_data/risk_scoring.py) 的 `TRIGGER_PATTERNS` 和 `_apply_level_floor` 中落地。

## 5. 混合赋权

默认权重先固定使用，后续清洗样本后再做混合赋权。

步骤：

1. 专家用 AHP 给出主观权重 `α_j`
2. 用清洗后的样本算熵权 `β_j`
3. 组合权重：

```text
w_j = sqrt(α_j × β_j) / Σ sqrt(α_j × β_j)
```

组合方法已在 [risk_scoring.py](/e:/风险预测和评估/risk_data/risk_scoring.py) 的 `combine_weights()` 和 `entropy_weight()` 中实现。

## 6. 数据清洗口径

当前样本不能直接作为分值真值，原因有三：

- 风险任务约 `300` 条，不是全部 `700` 条
- 其中约 `250` 条有精确分值，约 `50` 条只有分值区间
- 相同输入存在冲突标注，且当前没有 `L4` 样本

建议清洗规则：

- 重复输入但标签冲突时，先按“保守从高”形成候选分值
- 如文本里有明确伤害、聚集、舆情、公安等证据，以文本证据优先
- 区间样本只用于等级边界校准，不用于精确回归
- `L4` 先补专家样例，再进入训练或校准

数据分析脚本见 [analyze_risk_data.py](/e:/风险预测和评估/risk_data/analyze_risk_data.py)。
